Start building in 60 seconds

Three commands. Multiplayer-ready template. Auto-deploy on push.

1. Scaffold a new game

npx -y @progamestore/cli@latest init my-game

This creates a new game from a template with the SDK pre-configured, a working Durable Object skeleton for multiplayer, and compliance rules baked in.

2. Start developing

cd my-game
pnpm install
pnpm dev

Open http://localhost:5173. You get a working game shell with topbar, auth, and the useRooms() hook ready to wire up.

3. Publish to the store

pgs publish

The CLI runs compliance checks (build, brand, viewport, multiplayer correctness), provisions your GitHub repo + Cloudflare Worker + DNS + Stripe Connect subscription, and lists your game on the store.

What you get out of the box

  • GameShell + GameTopbar — brand-consistent UI shell, zero-scroll viewport lock
  • useRooms() — real-time multiplayer via Durable Objects (WebSocket)
  • useAuth() — platform-level authentication (sign in with GitHub)
  • useLeaderboard() — persistent D1 leaderboard
  • useGameSounds() — mute-aware sound effects
  • Cloud sync — 10MB/user, cross-device save state
  • Stripe subscription — you set the price, platform takes 10%
  • Auto-deploy — push to main, game updates live

Templates

The CLI offers multiple starting points:

  • Turn-based — chess, checkers, card games. Server validates moves, clients render.
  • Real-time — racing, fighting, co-op. 60fps state sync via WebSocket.
  • Persistent world — MMO-lite, sandbox. State lives on the server between sessions.

Use AI to build

Your scaffolded project includes a CLAUDE.md that points to the platform skills guide. Open any AI coding tool and describe your game:

# In Claude Code, Cursor, Codex, or any AI tool:
"Build a real-time racing game where 4 players compete on a procedural track.
Use the useRooms() hook for multiplayer. Canvas rendering."

The AI agent reads the skills guide and builds with the right SDK, the right patterns, and the right deploy flow.
