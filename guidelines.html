Platform Rules

These are non-negotiable. Every game on ProGameStore must follow these principles:

Free means free

No freemium, no ads, no plans, no paywalls, no "upgrade to unlock." If it's on ProGameStore, it stays free forever.

No tracking

Zero analytics, zero cookies, zero data collection. All player data stays on-device in localStorage.

Open source

MIT license. Public repo on the progamestore-online GitHub org. Anyone can read the code, verify privacy claims, and contribute.

Real games, not demos

Every game must be playable and complete. If it only shows a title screen, it's not ready.

Works everywhere

Responsive, mobile-first, PWA. Games must work on phones, tablets, and desktops.

Wordmark

The logo is a wordmark, not an icon. Written as one word with "Game" highlighted in the platform accent color:

Naming rules

Game Engines

Games use specialized engines depending on the game type. All integrate with the React + Vite stack.

KaplaySimple, fun 2D gamestemplate-game-kaplay
HTML5 CanvasDirect canvas drawing for classic 2D gamestemplate-game-canvas

2D games

For 2D games, use HTML5 Canvas directly or a framework like Phaser, PixiJS, or Kaplay. The canvas template gives you a bare <canvas> element with a game loop. Phaser and PixiJS templates include their respective engines pre-configured.

3D games

For 3D games, use Three.js (via React Three Fiber) or Babylon.js. The 3D template includes Three.js with R3F, a basic scene, camera controls, and lighting. Babylon.js template includes the engine with a physics-ready scene.

Publishing Process

How games get deployed

  1. Push to main — creator pushes code to the GitHub repo
  2. Cloudflare Pages builds — automatic build triggered on push (runs pnpm install && pnpm build)
  3. Deploy to edge — built assets deployed to Cloudflare's global CDN
  4. Live on subdomain — available at gamename.progamestore.online

No CI/CD configuration needed. No tokens to manage. Cloudflare Pages is connected directly to GitHub — push and it deploys.

How a new game gets listed

  1. Apply — Open an issue on submissions with your game idea and GitHub username
  2. Review — Reviewed within 48 hours
  3. Provision — Repo created in org, CF Pages project set up, subdomain assigned
  4. Build — Develop the game following these guidelines
  5. Listed — Game card added to the ProGameStore landing page

Compliance Checklist

Every game must pass ALL of these before listing:

Gameplay

  • Game is playable and complete (not a demo or placeholder)
  • Works on mobile (320px+) with touch controls
  • Works on desktop (1024px+) with keyboard/mouse
  • Saves progress/high scores locally (localStorage)
  • No console errors in production build

Design

  • Uses shared CSS custom properties for UI chrome
  • Emerald accent color via var(--accent)
  • Supports dark mode via system preference
  • Responsive layout — works across screen sizes

PWA

  • Has manifest.json with name, icons, theme color
  • Includes apple-mobile-web-app-capable meta tag
  • Registers a service worker
  • Works when added to home screen

Privacy

  • Zero network requests for tracking/analytics
  • No cookies
  • No external scripts (CDN fonts are OK)
  • All player data in localStorage only

Performance

  • First load <2s on 3G
  • Smooth gameplay at 60fps on mid-range devices
  • No layout shift after load

Code

  • TypeScript with strict mode
  • No any types
  • Clean build with zero warnings
  • MIT license in repo
