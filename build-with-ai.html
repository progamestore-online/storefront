Build a multiplayer game with AI.

ProGameStore is AI-first. Drop our skills.md into your AI tool of choice, and the agent already knows the SDK, the multiplayer patterns, the compliance rules, and the deploy flow before it writes a line of code.

The workflow

  1. Scaffold: npx -y @progamestore/cli@latest init my-game
  2. Open your AI tool in the project directory
  3. Describe your game. The AI reads CLAUDE.md which points to the platform guide.
  4. Build + iterate. The AI uses @progamestore/games for the shell and useRooms() for multiplayer.
  5. Publish: pgs publish

Claude Code

Anthropic's terminal CLI. The scaffolded project includes CLAUDE.md that Claude reads automatically.

# Scaffold and open
npx -y @progamestore/cli@latest init my-racing-game
cd my-racing-game
claude

# Then describe your game:
"Build a real-time 4-player racing game. Canvas rendering, procedural tracks,
useRooms() for multiplayer. Include boost pads and collision physics."

Cursor

AI-native VS Code fork. Add the skills guide to your context.

# Scaffold
npx -y @progamestore/cli@latest init my-card-game
cd my-card-game
cursor .

# In Cursor chat, add context:
# @file CLAUDE.md
# Then describe your game.

Codex CLI

OpenAI's open-source terminal agent.

npx -y @progamestore/cli@latest init my-puzzle-game
cd my-puzzle-game
codex

# Codex reads CLAUDE.md from the project root.

Any other AI tool

For tools that don't auto-read project files, paste the skills guide URL:

Read https://progamestore.online/skills.md for platform context.
Then build a turn-based strategy game with 2-player multiplayer.

What the AI knows from skills.md

  • SDK components: GameShell, GameTopbar, GameAuth, GameButton
  • Hooks: useRooms(), useAuth(), useLeaderboard(), useGameSounds()
  • Multiplayer patterns: turn-based, real-time, persistent world
  • Compliance rules: brand fonts, viewport lock, no tracking
  • Deploy flow: pgs publish provisions everything
  • Stripe Connect: developer sets price, platform takes 10%

Example prompts

Turn-based

"Build a multiplayer checkers game. Server validates moves. useRooms() for real-time sync. Include an AI opponent using minimax for single-player mode."

Real-time

"Build a 4-player arena shooter. Canvas rendering at 60fps. useRooms() broadcasts position + projectiles. Server authoritative for hit detection."

Persistent world

"Build a farming sim where plots persist between sessions. useRooms() for visiting friends' farms. Cloud sync for inventory. Cron worker for crop growth."
